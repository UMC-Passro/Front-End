# 패스로(Passro)

## 1. 프로젝트 소개

본 프로젝트는 대학생이 매일 오가는 통학 경로를 활용해 소형 택배를 대신 배송해주는 P2P 배송 플랫폼을 목표로 하는 웹 서비스입니다.

사용자가 배송 요청 등록, 운송 가능 경로 확인, 매칭, 배송 진행 상태 확인 등의 핵심 기능을 보다 쉽고 편리하게 이용할 수 있도록 프론트엔드와 백엔드를 분리하여 개발합니다.

```bash

└── dev

```

- `main`: 최종 배포용 브랜치

- `feature`: 기능 단위 작업 브랜치

## 6. 커밋 컨벤션

```bash
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서 수정
style: 코드 포맷팅
refactor: 코드 리팩토링
chore: 기타 설정 변경
```

예시:

```bash
feat: 로그인 페이지 UI 구현
fix: 라우팅 오류 수정
docs: README 실행 방법 추가
```

## 7. PR 컨벤션

### PR 제목 양식

```bash
[type] 작업 내용 요약
```

- `type`은 커밋 컨벤션의 `feat`, `fix`, `docs`, `style`, `refactor`, `chore` 중 하나를 사용합니다.
- 제목은 PR에서 변경한 핵심 내용을 한 문장으로 간결하게 작성합니다.

예시:

```bash
[feat] 배송 요청 등록 페이지 UI 구현
[fix] 배송 상태 라우팅 오류 수정
[docs] README 화면 플로우 추가
```

### PR 본문 양식

PR 작성 시 다음 내용을 포함합니다.

```md
## 수정자
- 이름 또는 GitHub ID

## 작업 내용
- 구현한 기능 설명

## 변경 사항
- 수정된 주요 파일 또는 구조

## 확인 사항
- 실행 여부
- 에러 여부
- 추가 확인이 필요한 부분
```

## 9. 화면 목록 및 플로우

### 화면 목록

- 메인 페이지
- 로그인 페이지

- 배송 완료 페이지
- 마이페이지

### 기본 플로우

```bash
회원가입 / 로그인
└── 배송 요청 등록
    └── 경로 유사도 기반 배송자 자동 탐색
        └── 배송자 후보 확인
            └── 매칭 요청
                └── 매칭 확정 및 푸시 알림
                    └── 배송 상태 단계별 업데이트
                        └── 배송 완료 확인
                            └── 포인트 자동 정산
                                └── 리뷰 / 평점 작성
```

### MVP 기준 핵심 흐름

1. 발송자는 출발지, 도착지, 물품 크기, 사진을 포함해 배송 요청을 등록합니다.
2. 시스템은 등록된 배송 요청과 배송자의 통학 경로를 비교해 유사도가 높은 배송자를 자동으로 탐색합니다.
3. 발송자는 추천된 배송자 후보를 확인하고 매칭 요청을 보냅니다.
4. 배송자가 요청을 수락하면 매칭이 확정되고 발송자에게 푸시 알림이 전달됩니다.
5. 배송자는 픽업, 이동 중, 배송 완료 등 배송 상태를 단계별로 업데이트합니다.
6. 배송 완료가 확인되면 포인트가 자동 정산됩니다.
7. 배송 완료 후 발송자는 리뷰와 평점을 남길 수 있습니다.
